# Search Screen Implementation Plan

## Summary

Build a Search screen with a 2x3 grid of compact cards (6 per page, no scrolling), advanced filter bottom sheet with ~20 filter options, detail page with Like + Chat buttons (no Pass), shared daily limits with Discover, and filter persistence via SharedPreferences.

## Step 1: SearchService (`lib/services/search_service.dart`)

Static service class following `DiscoverService` pattern.

```dart
class SearchService {
  static Future<Response> search({...all params...}) async {
    // Build params map, only include non-null values
    // Use ApiService.noCache for real-time results
    return await ApiService.get('/search', queryParams: params, cacheOptions: ApiService.noCache);
  }
  
  static Future<Response> swipeUser(String userId, String direction) async { ... }
  static Future<Response> sendFirstMessage(String userId, String content) async { ... }
  static Future<Response> getMyLimits() async { ... }
}
```

Key: swipeUser/sendFirstMessage/getMyLimits are identical to `DiscoverService` methods. Reuse the same API endpoints (`/swipes`, `/messages/$userId/text`, `/rewards/my-limits`).

### Key Methods

- `loadProfiles()` — resets offset to 0, calls `SearchService.search(limit: 6, offset: 0, ...all filters)`, parses response, calls `_refreshLimits()`
- `loadMore()` — loads next page (offset += 6), replaces current cards (page-replace mode, not append)
- `likeUser(profile)` — checks `isLikeBlocked`, POST /swipes, removes from list, refreshes limits
- `chatWithUser(profile, {message})` — checks limits, POST /swipes + optional POST /messages, removes from list, refreshes limits
- `_refreshLimits()` — GET /rewards/my-limits (same endpoint as DiscoverProvider)
- `_loadFilters()` / `_saveFilters()` — SharedPreferences with `search_` prefix keys
- `resetFilters()` — resets all to defaults, clears prefs, reloads

### Computed Getters

```dart
bool get isLikeBlocked => !_isPremium && _likesRemaining <= 0;
bool get isChatBlocked => !_isPremium && _chatsRemaining <= 0;
int get totalPages => (_total / _pageSize).ceil();
bool get canLoadNext => _currentPage < totalPages - 1;
bool get canLoadPrev => _currentPage > 0;
bool get hasActiveFilters => /* check all non-default values */;
int get activeFilterCount => /* count active filters */;
```

### Filter Quick Bar

Horizontal `SingleChildScrollView` with filter chips. Each chip shows current value and opens a picker:
- Gender → bottom sheet with ChoiceChips
- Age → bottom sheet with RangeSlider
- Distance → bottom sheet with Slider
- Sort → bottom sheet with ChoiceChips
- Advanced → opens `SearchFilterSheet`

### Page Navigation

Bottom bar with:
- "Page {current+1} of {totalPages}" centered text
- Row of [Previous] and [Next] buttons
- Previous disabled when `currentPage == 0`
- Next disabled when `currentPage >= totalPages - 1`
- Tapping Next calls `provider.loadPage(currentPage + 1)`
- Tapping Prev calls `provider.loadPage(currentPage - 1)`

### States

- **Loading**: 6 shimmer cards in grid layout
- **Empty**: Illustration + "No results found" + "Try adjusting your filters" + refresh button
- **Error**: Cloud icon + error message + retry button
- **Content**: Grid of 6 cards + page navigation

## Step 5: SearchFilterSheet (`lib/screens/search/search_filter_sheet.dart`)

`showModalBottomSheet` with `isScrollControlled: true` (full-height sheet).

### Sections (top to bottom, all in a scrollable ListView)

1. **Header**: "Advanced Filters" title + drag handle + Reset button
2. **Location** (English):
   - Country dropdown (from `LocationService.getCountries()`)
   - Province dropdown (dependent, from `LocationService.getStates()`)
   - City dropdown (dependent, from `LocationService.getCities()`)
3. **Basic**:
   - Age Range: RangeSlider (18-100)
   - Distance: Slider (1-500 km, optional)
   - Gender: ChoiceChip row (All/Male/Female)
   - Sort: ChoiceChip row (Recent/Distance/Age/Name)
4. **Physical**:
   - Height Range: RangeSlider (140-220 cm, optional)
   - Weight Range: RangeSlider (40-150 kg, optional)
   - Body Type: ChoiceChip row (slim/average/athletic/curvy/muscular/overweight)
5. **Lifestyle**:
   - Relationship Status: ChoiceChip (single/divorced/widowed/separated)
   - Education: ChoiceChip (high_school/bachelor/master/phd)
   - Smoking: ChoiceChip (never/occasionally/regularly)
   - Drinking: ChoiceChip (never/socially/regularly)
   - Political: ChoiceChip (liberal/conservative/moderate/apolitical)
   - Children: ChoiceChip (have/dont_have/want/dont_want)
   - Living: ChoiceChip (alone/with_family/with_roommate/with_partner)
6. **Background**:
   - Religion: text input or preset chips
   - Ethnicity: text input or preset chips
7. **Interests**:
   - Multi-select chips grouped by category (from `OnboardingService.getInterests()`)
   - Each chip shows emoji icon + name
8. **Languages**:
   - Multi-select chips (common languages: fa, en, ar, tr, fr, de, etc.)
9. **Verification**:
   - Has Photos toggle
   - Verified Only toggle
10. **Bottom**:
    - "Reset All" text button
    - "Apply Filters" primary button (closes sheet, applies filters to provider)

### State Management

The sheet uses `StatefulBuilder` for local state. On "Apply", it calls `provider.setAllFilters(...)` which updates all filter fields at once, saves to prefs, and reloads profiles.

## Step 6: SearchProfileDetail (`lib/screens/search/search_profile_detail.dart`)

Adapted from `ProfileDetailScreen` with two key differences:
1. **Only 2 buttons**: Like (heart) + Chat (bubble) — NO Pass/X button
2. **Callbacks**: Uses `SearchProvider.likeUser()` and `SearchProvider.chatWithUser()`

### Layout (same as ProfileDetailScreen)

- CustomScrollView with hero image, photo strip, body sections (bio, physical, lifestyle, background, languages, interests, prompts)
- Bottom action bar: Like button (gradient, badge count) + Chat button (outlined, badge count)

### Action Flow

**Like button:**
1. Check `provider.isLikeBlocked` → show limit dialog if blocked
2. Call `provider.likeUser(profile)`
3. If matched → show match dialog
4. Pop screen, return to search grid (next page loads automatically)

**Chat button:**
1. Check `provider.isChatBlocked` → show limit dialog if blocked
2. Show bottom sheet with TextField (same as discover's chat sheet)
3. Call `provider.chatWithUser(profile, message: message)`
4. If matched → show match dialog
5. Pop screen

### Match Dialog

Same as discover's match dialog — heart icon, "It's a Match!", subtitle, "Send a Message" / "Continue Browsing" buttons.
