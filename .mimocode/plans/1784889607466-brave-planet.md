# Discover UI: Badges, Popup Limits, Detail Navigation

## Changes

### 1. Remove limit progress bars
- Delete `_buildLimitIndicator` and `_buildSingleLimitBar` methods
- Remove `_buildLimitIndicator(provider, isDark, primaryColor)` call from `build()`

### 2. Add badge count on like and message buttons
- Add `badgeCount` parameter to `DiscoverActionButton` in `discover_action_button.dart`
- Show a small red circle with the remaining count at top-right of the button (like iOS notification badges)
- Pass `provider.likesRemaining` and `provider.chatsRemaining` from `_buildActionButtons`
- Hide badge when premium or count is 0

### 3. Limit reached → popup (not snackbar)
- Replace `_showLimitReached` snackbar with `showDialog` AlertDialog
- Show title + message + "OK" button
- Use existing ARB keys `discover_limit_reached_likes` / `discover_limit_reached_chats`

### 4. Detail page → like/pass closes and returns to discover
- In `_openProfileDetail`, the callbacks already call `_handleSwipeRight`/`_handleSwipeLeft`
- After the action completes, pop the detail screen: wrap callbacks to ` Navigator.pop` after the action
- Pass a callback that also pops the detail page context
